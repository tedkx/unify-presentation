Part 4Data Flow

Using core

  • Mechanism combining react-redux connect, reducers & sagas
  • Common api calls simplification
  • Uses service instructions

Core Connect()

  • Supplies call() method for calling services
  • Supplies a triplet of service-related props
  • Keeps all data on the core reducer (can be changed)
  • Usage 
    connect(Component, [serviceInstructions], reducer, ...reduxConnectProps)

Core Connect: Component Props

  • call / canCall 
    if(this.props.canCall('myService')) {
  this.props.call('myService')
}
  • Triplet of props: pending / error 
    render() {
  const { 
    myService, 
    myServicePending, 
    myServiceError 
  } = this.props;
}
  • clear 
    this.props.clear('myService'))

Core Connect()

  • Connected Component 
    import { connect } from @eurobank-unify/core;
import MyComponent from 'my-component';

connect(MyComponent, ['usersList'])
  • Component 
    import React from 'react';

class MyComponent extends React.Component {
  componentDidMount() {
    this.props.call('usersList');
  }
  componentWillUnmount() {
    this.props.clear('usersList');
  }
  render() {
    return this.props.usersListPending
      ? 
    Loading
    
      : 
    { this.props.usersList.map(u => {u.username})}
    
  }
}

Service instructions

Make a call to apiRoute, store the response on reducerProperty in the core reducer and make it available to my component through componentProperty prop

  • React component property
  • Api endpoint
  • Reducer property

Service instructions: Initialization

  • One string to rule 'em all 
    const serviceInstruction = 'userList'
// OR
import { createServiceInstruction} from @eurobank-unify/core;
const serviceInstruction = createServiceInstruction('userList')
  • Customizations through second parameter 
    import { createServiceInstruction} from @eurobank-unify/core;
const serviceInstruction = createServiceInstruction('userList', {
  componentProperty: 'myUsers',
  apiRoute: 'fetchMyUsers'
})
  • Reducer property
